# MLCOE AKGEC Wildcard Tasks
Welcome to the MLCOE AKGEC Wildcard Tasks Repository.

This repository contains the wildcard tasks for the MLCOE AKGEC recruitment process. There are tasks available across three domains: Machine Learning, Web Development, and Design. Each domain has its own dedicated PDF task file. Out of the tasks provided, you are required to attempt and submit only one.

## Repository Structure
The repository is structured as follows:

```text
/
├── Datasets/               # Contains the datasets required for the tasks
├── MLCOE_ML_Wildcard_Recruitment_Task.pdf    # Task details and requirements for Machine Learning
├── MLCOE_Web_Development_Recruitment_Task.pdf     # Task details and requirements for Web Development
└── Design.pdf              # Task details and requirements for Design
```
Please open the PDF corresponding to your chosen domain to view the problem statements. If your task requires a dataset, you can access them via the Google Drive links below:

## Submission Process
All wildcard submissions for **Machine Learning, Web Development, and Design** must be submitted via the official Form linked at the bottom of this page.

Before filling out the form, please ensure you have prepared your submission according to your domain requirements:

### Machine Learning & Web Development
* Upload your complete project to a public GitHub repository under your own account.
* Ensure your repository includes a clear `README.md` explaining how to run the project.
* For Web Development, providing a live hosted link (e.g., Vercel, Netlify, GitHub Pages) is highly encouraged.

### Design
* Host your design on Figma, Behance, or a cloud drive (ensure the link is set to "Anyone with the link can view").
* Alternatively, prepare a PDF or image export of your design solution to upload or link.

---

## Submission Guidelines
* Read the task descriptions in your domain's PDF carefully before starting.
* Make sure your GitHub repositories or design links are set to **public** so they can be accessed by the evaluators.
* Include setup, dataset path configuration, or installation instructions in your project documentation if applicable.
* Double-check that all required deliverables mentioned in the PDF are completed before submitting the form.

## Evaluation
Submissions will be evaluated based on:
* **Completeness & Correctness**
* **Code/Design Quality**
* **Project Structure & Organization**
* **Documentation & Presentation**

*Meeting only the minimum requirements does not guarantee selection.*
